# 프로젝트 회고

## 1. 잘한 점

이 프로젝트에서 가장 좋았던 점은, 데모 수준이었더라도 **헤드셋 내부에 필요한 부품이 비교적 깔끔하게 들어가도록 구성할 수 있었다는 점**이다.  
단순히 아이디어 문서로 끝난 것이 아니라, 실제 장치 형태를 갖춘 프로토타입으로 정리할 수 있었던 점이 의미 있었다.

또한 음성 인식과 STT 기반 처리 흐름도 **테스트 범위 내에서는 비교적 정상적으로 동작**했다.  
생활 소리를 입력으로 받아 텍스트 변환과 후속 처리로 이어지는 구조를 실제로 시도해보면서,  
프로젝트의 핵심 아이디어를 데모 가능한 수준까지 연결해본 경험이 남았다.

---

## 2. 어려웠던 점

가장 어려웠던 부분은 **라즈베리파이 계열 보드와 ReSpeaker 마이크 모듈을 연결하고, 이를 기반으로 STT 처리까지 이어가는 과정**이었다.

당시에는 라즈베리파이와 Linux 계열 환경을 처음 다뤄보는 수준이었기 때문에,  
장치 연결 자체뿐 아니라 오디오 입력과 사운드 관련 설정 문제에서도 시행착오가 많았다.  
특히 음성 입력 장치 인식이나 STT 처리 과정에서 바로 원하는 결과가 나오지 않았고,  
환경 설정과 장치 연동 문제를 하나씩 직접 찾아가며 해결해야 했던 점이 가장 큰 난관이었다.

또 하나의 어려움은, 지금과 달리 당시에는 AI 도구의 도움을 크게 기대하기 어려웠다는 점이다.  
그래서 대부분의 자료를 직접 찾아보고, 필요한 코드를 직접 작성하고 수정하면서 진행해야 했다.  
이 과정은 시간이 오래 걸렸지만, 반대로 임베디드·리눅스 환경과 장치 연동 문제를 스스로 파고들어 해결해보는 경험으로 남았다.

---

## 3. 배운 점

이 프로젝트를 통해 단순히 코드를 작성하는 것과,  
**실제 장치 환경에서 소프트웨어가 동작하도록 연결하는 것 사이에는 큰 차이가 있다**는 점을 배웠다.

특히 다음과 같은 점을 체감했다.

- 모듈 선택과 장치 간 연결 방식이 프로젝트 전체 난이도에 큰 영향을 준다.
- 음성 처리 프로젝트에서는 STT 로직 자체뿐 아니라 입력 장치, 사운드 환경, OS 설정이 매우 중요하다.
- 임베디드/리눅스 환경에서는 라이브러리나 설정 문제를 직접 확인하고 해결하는 능력이 필요하다.
- 데모 수준의 결과라도 실제로 끝까지 연결해보는 경험이 중요하다.

즉, 이 프로젝트는 단순히 기능 하나를 구현한 경험보다,  
**하드웨어와 소프트웨어가 만나는 지점에서 발생하는 문제를 직접 다뤄본 경험**이라는 점에서 의미가 있었다.

---

## 4. 개선하고 싶은 점

다시 이 프로젝트를 진행한다면 다음 부분을 보완하고 싶다.

### 4-1. 헤드셋 마감 품질 개선
프로토타입 단계에서는 기능 연결 자체에 집중했기 때문에,  
외형 마감이나 완성도 측면에서는 아쉬움이 남는다.  
다시 만든다면 내부 부품 고정 방식, 외형 마감, 전체 착용감 등을 더 정교하게 다듬고 싶다.

### 4-2. STT 적용 범위 확대
당시에는 테스트 범위 안에서 동작 확인에 의미가 있었지만,  
실제 일상생활에서는 훨씬 더 다양한 환경과 소음 상황이 존재한다.  
따라서 다시 진행한다면 특정 상황에 한정된 데모를 넘어서,  
**보다 다양한 생활 환경에서도 안정적으로 동작할 수 있도록 STT 처리 범위를 넓히고 싶다.**

---

## 5. 이후 확장에 대한 생각

현재 기준으로는 이 프로젝트를 구체적인 후속 서비스나 제품 형태로 확장하는 아이디어를 명확히 정리해두지는 않았다.  
다만, 생활 소리를 의미 단위로 판별하고 사용자에게 전달하는 접근 자체는  
보조기기나 일상 보조 시스템 측면에서 충분히 발전 가능성이 있는 방향이라고 생각한다.

---

## 6. 정리

이 프로젝트는 최종 완성형 제품을 만든 경험이라기보다,  
**아이디어를 실제 장치 형태와 소프트웨어 흐름으로 연결해본 프로토타입 경험**에 가깝다.

특히 ReSpeaker와 라즈베리파이 계열 보드, STT 처리, 앱 연동까지 이어지는 구조를 직접 다뤄보면서  
임베디드 환경의 제약과 소프트웨어 연결의 난점을 몸으로 배울 수 있었다.

결과적으로 이 프로젝트는 수상과 전시라는 결과뿐 아니라,  
**처음 접하는 환경에서 장치 연동과 음성 처리 문제를 끝까지 구현해본 경험**이라는 점에서 의미가 있었다.
