# 나의 역할

이 문서는 **청각장애인을 위한 헤드셋형 보조기기** 프로젝트에서 내가 맡았던 역할을 정리한 문서이다.  
현재 남아 있는 자료는 상장, 판넬, 행사 사진, 내부 부품 사진, 실물 시연 영상, 일부 organization 저장소 링크이며, 최종 통합 코드베이스는 보존되지 않았다.  
따라서 이 문서는 **현재 남아 있는 자료와 내가 설명 가능한 범위**를 기준으로 작성한다.

---

## 1. 역할 요약

나는 이 프로젝트에서 **소프트웨어 구현 전반을 중심으로 참여한 팀원**이었다.

특히 다음 영역에 주로 기여했다.

- 음성 입력 장치와 제어 보드 간 구성 검토 및 연결 방식 정리
- STT 기반 음성 처리 로직 설계 및 구현
- 장치와 앱 사이의 통신 흐름 구성
- Android 앱 작업 보조

하드웨어 자체를 직접 제작하는 역할보다는,  
**어떤 모듈을 사용할지 선택하고, 장치 간 통신과 소프트웨어 흐름을 연결하는 역할**에 더 가까웠다.

---

## 2. 내가 주로 맡았던 작업

## 2-1. 모듈 및 보드 구성 검토
프로젝트 초기에 음성 입력 장치와 제어 보드 조합을 검토했고,  
Seeed Studio ReSpeaker Mic Array v2.0 모듈과 라즈베리파이 계열 보드 중 호환 가능한 구성을 선택하는 데 참여했다.

현재 기억 기준으로는 Raspberry Pi Pico 또는 Zero 2 W 계열을 검토했던 것으로 기억하지만,  
공개 자료와 내부 사진 기준으로는 **Pico 계열 중심으로 정리하는 것이 더 안전하다.**

즉, 내 역할은 단순 하드웨어 조립보다도  
**어떤 보드/모듈 조합이 실제 구현에 적합한지 판단하고 연결 구조를 잡는 쪽**에 가까웠다.

---

## 2-2. 장치 간 통신 흐름 구성
내가 맡은 핵심 역할 중 하나는  
마이크 입력 장치, 제어 보드, 앱 사이의 **통신 흐름을 연결하는 작업**이었다.

기억 기준으로는 다음과 같은 형태를 중심으로 작업했다.

- 장치 간 SSH 기반 연결 또는 네트워크 연결 구성
- Bluetooth 또는 무선 연결 방식 검토 및 적용
- 마이크 입력 장치와 앱 사이에서 상태 또는 결과를 주고받는 흐름 정리

정확한 프로토콜 세부 구현은 현재 모두 보존되어 있지 않지만,  
내가 메인으로 맡았던 것은 **장치 간 연결과 통신 경로를 실제 동작 가능한 구조로 만드는 일**이었다.

---

## 2-3. STT 음성 처리 로직 구현
프로젝트에서 내가 가장 중심적으로 맡았던 부분은  
**음성 데이터를 텍스트로 변환하고, 이를 후속 처리에 연결하는 로직**이었다.

내가 담당한 작업은 다음과 같다.

- 마이크 입력 음성 처리 흐름 구성
- STT 처리 로직 구현 또는 연결
- 텍스트 결과를 기반으로 후속 동작과 연결되는 소프트웨어 흐름 정리
- 전체 시스템에서 음성 인식 이후 단계가 끊기지 않도록 로직 중심축 구성

즉, 하드웨어를 만드는 역할보다  
**음성 입력 이후 소프트웨어적으로 어떻게 처리되고 전달될지를 구현하는 역할**을 주로 맡았다.

---

## 2-4. Android 앱 작업 보조
나는 메인 역할이 앱 단독 개발자는 아니었지만,  
프로젝트 진행 과정에서 **Android 앱 작업에도 보조적으로 참여했다.**

주로 다음과 같은 맥락이었다.

- 장치와 앱 사이의 연동 관점에서 앱 작업 지원
- 상태 표시 또는 결과 전달 흐름 확인
- 전체 사용자 흐름이 끊기지 않도록 앱 측 작업 보조

따라서 이 프로젝트에서 나는  
임베디드 장치 쪽 소프트웨어와 앱 사이를 이어주는 역할도 함께 수행했다.

---

## 3. 내가 이해하고 있던 전체 시스템 방향

내가 기억하는 프로젝트의 전체 작동 흐름은 아래와 같았다.

1. 음성인식 모듈이 생활 속 소리를 인식
2. STT 변환 후 DB를 조회
3. 키워드를 인식하고 중요도를 판별
4. 판별 결과를 앱으로 전송
5. 중요도에 따라 사용자에게 알림

즉, 프로젝트의 핵심은 단순 음성 인식 자체보다도  
**생활 소리를 텍스트로 변환한 뒤, 이를 DB 및 키워드 판별 로직과 연결하고,  
최종적으로 앱과 사용자 알림까지 이어지도록 만드는 흐름**에 있었다.

내 역할은 이 전체 흐름 중에서도 특히 다음 구간에 집중되어 있었다.

- STT 변환 이후의 소프트웨어 처리
- DB 조회 및 키워드 판별 로직 연결
- 앱으로 결과를 전달하는 통신 흐름 구성
- Android 앱 연동 보조
